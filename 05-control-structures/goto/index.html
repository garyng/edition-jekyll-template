Control Structures

goto

The goto statement transfers the control of the program immediately to a labeled statement (Docs, 2015).

Even though goto is useful in jumping out of a deeply nested loop, the use of goto statement is highly discouraged as it often results in spaghetti code:

…widespread and unconstrained use of GOTO statements has led to programmers producing inconsistent, incomplete and generally unmaintainable programs.
Such code is often known as ‘spaghetti’, given its convoluted and tangled control structure.

(Cram & Hedley, 2005)

The previous nested for loop

for (int i = 0; i < 10; i++)
{
    for (int j = 0; j < 10; j++)
    {
        if (j > i)
        {
            break;
        }
        Console.Write($"{j}");
    }
    Console.WriteLine();
}

can be written using goto as

for (int i = 0; i < 10; i++)
{
    for (int j = 0; j < 10; j++)
    {
        if (j > i)
        {
            goto outer;
        }
        Console.Write($"{j}");
    }
outer:
    Console.WriteLine();
}

Note that we need a label (outer: in the example above) to indicate the position we want the program control to go.
